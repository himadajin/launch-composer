# Launch Composer - 拡張機能ホスト仕様

`launch-composer`（extension）パッケージの Extension Host 側が実装する仕様。

前提として読むべきファイル:

- [spec.md](./spec.md) §4 — スキーマ定義（JSON ファイル形式）
- [spec-communication.md](./spec-communication.md) §1 — Webview との通信契約

---

## 概要

Extension Host は Launch Composer の VSCode 統合層であり、ユーザーの設定ファイルと VSCode API を接続する役割を担う。

ユーザーが Generate を実行すると、Extension Host は `.vscode/launch-composer/` 以下の JSON ファイルを読み込み、`@launch-composer/core` の生成関数を呼び出してマージ・バリデーションを行い、結果を `.vscode/launch.json` に書き込む。GUI 操作（サイドバーの TreeView・編集フォームの Webview Panel）はすべて Extension Host が管理し、ファイルの変更を `FileSystemWatcher` で監視してサイドバーを自動更新する。

`.vscode/launch-composer/`, `templates/`, `configs/` は事前に存在している必要はない。読み取り時に未作成であれば正常な空状態として扱い、ファイル作成や保存などの書き込み時に必要なディレクトリを自動作成する。

このファイルは以下を仕様として定める:

- `launch.json` の生成動作（上書きルール・確認ダイアログ・出力フォーマット）
- ファイル監視の動作
- コマンドパレットおよびサイドバーへのコマンド登録
- 拡張機能の設定項目（`settings.json`）

---

## 1. launch.json 生成動作

Generate 実行時の `.vscode/launch.json` の生成に関する動作を定める。

### 1.1 launch.json の上書き動作

Generate を実行すると、既存の `.vscode/launch.json` の内容をすべて破棄して新しい内容で上書きする。手動で追加した構成やコメントも含め、ファイルの内容は完全に置き換わる。launch.json の一部だけを更新する部分管理は行わない。

enabled な config エントリが 0 件の場合、またはテンプレートが 1 件も存在しない場合は、警告なしで `configurations: []` を持つ launch.json を生成する。

`.vscode/launch-composer/`, `templates/`, `configs/` が未作成であっても Generate の前提エラーにはしない。この場合はテンプレート 0 件、config 0 件として扱い、空の `launch.json` を生成する。

既存の `.vscode/launch.json` が存在しない場合も Generate の前提エラーにはしない。上書き確認は表示せず、新規作成としてそのまま書き込む。

Generate は `type` / `request` の未設定だけを理由に失敗しない。最終的な configuration にこれらの値がない場合でも、出力時に両キーを空文字 `""` で補完して書き込む。

### 1.2 上書き確認ダイアログ

既存の launch.json がある場合、Generate のたびに確認ダイアログを表示する。

```
⚠ launch.json will be overwritten. Continue?
  [Yes]  [Cancel]
  ☐ Don't ask again
```

- 「Don't ask again」にチェックを入れて Yes を選択した場合、`settings.json` に `launch-composer.confirmOverwrite: false` を保存し、以降は確認なしで上書きする。
- ダイアログは `vscode.window.showWarningMessage` で実装する。

### 1.3 生成時に挿入する自動生成コメント

生成する launch.json の先頭に、自動生成されたファイルであることを示すコメントを挿入する。launch.json は jsonc（コメント付き JSON）形式として出力する。

```jsonc
// This file is auto-generated by Launch Composer.
// Do not edit manually. Changes will be overwritten.
{
  "version": "0.2.0",
  "configurations": [ ... ]
}
```

---

### 1.4 未作成ディレクトリの扱い

Extension Host は Launch Composer 用ディレクトリが未作成であることをエラーや警告にしない。未作成時の動作は以下のとおり。

| 状態                              | 挙動                                     |
| --------------------------------- | ---------------------------------------- |
| `.vscode/launch-composer/` がない | templates/configs ともに空として扱う     |
| `templates/` のみない             | templates は空、configs は存在分だけ読む |
| `configs/` のみない               | configs は空、templates は存在分だけ読む |

未作成ディレクトリの扱いは読み取り系 API 全体で一貫させる。TreeView 表示、Generate、テンプレート選択候補の取得などは、いずれも空データとして振る舞う。

読み取り対象の JSON ファイルが一覧取得後に削除されるなど、読み込み時点で未存在になっていた場合も、読み取り系 UI はそのファイルを欠損扱いとして無視し、未存在そのものをユーザー向けエラーにはしない。

## 2. ファイル監視

Extension Host は設定ファイルの変更をリアルタイムで検知し、サイドバーを常に最新の状態に保つ。`vscode.workspace.createFileSystemWatcher` を使い、`.vscode/launch-composer/**/*.json` のファイル変更を監視する。

| イベント                   | 動作                                             |
| -------------------------- | ------------------------------------------------ |
| ファイルの作成・変更・削除 | サイドバーの TreeView を自動的にリフレッシュする |

エディタパネル（Webview）には外部変更を通知しない。再度アイテムをクリックして開けば最新のデータが読み込まれる。

## 3. コマンド登録

拡張機能が VSCode に登録するコマンドの一覧を定める。コマンドパレットに表示するコマンドと、TreeView 専用のコマンドに分類される。

### 3.1 コマンドパレットに表示するコマンド

ユーザーがコマンドパレットから直接呼び出せるコマンドは以下のとおり。

| コマンド ID                   | 表示名                                | トリガー                                             |
| ----------------------------- | ------------------------------------- | ---------------------------------------------------- |
| `launch-composer.generate`    | Launch Composer: Generate launch.json | コマンドパレット、CONFIGS ビューヘッダーの `$(play)` |
| `launch-composer.init`        | Launch Composer: Initialize           | コマンドパレット                                     |
| `launch-composer.addTemplate` | Launch Composer: Add Template         | コマンドパレット                                     |

`launch-composer.init` は必須セットアップではなく、Launch Composer を使い始めるための保存先と雛形ファイルを先に揃えたい場合の任意の補助コマンドとする。未実行でも Generate、TreeView 表示、各種追加操作は通常どおり利用できる。

`launch-composer.init` はリセットではなく、不足している既定の保存先と雛形を補うコマンドとする。既存のファイル・ディレクトリは一切変更せず、存在しないものだけを作成する。作成対象:

1. `.vscode/launch-composer/`
2. `.vscode/launch-composer/templates/`
3. `.vscode/launch-composer/configs/`
4. `.vscode/launch-composer/templates/template.json`
5. `.vscode/launch-composer/configs/config.json`

`template.json` と `config.json` は JSONC コメント付きの空配列を初期内容とする。これらのファイルがすでに存在する場合、その内容の確認・追記・上書きは行わない。

処理完了後、通知バーで保存先が利用可能であり、不足していた既定ファイルが作成された場合はそのことも簡潔に報告する。

## 4. 設定項目（settings.json）

ユーザーが `settings.json` で設定できる拡張機能固有の設定項目は以下のとおり。

| 設定キー                           | 型      | デフォルト | 説明                                                                                                   |
| ---------------------------------- | ------- | ---------- | ------------------------------------------------------------------------------------------------------ |
| `launch-composer.confirmOverwrite` | boolean | `true`     | `true` の場合、Generate 実行時に上書き確認ダイアログを表示する。`false` の場合は確認なしで上書きする。 |
| `launch-composer.autoSaveDelay`    | number  | `1000`     | `<TextInput>` フィールド変更後、JSON ファイルへの書き込みまでの debounce 遅延時間（ミリ秒）。          |
